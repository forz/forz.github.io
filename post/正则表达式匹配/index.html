LeetCode 44. Wildcard Matching#

Implement wildcard pattern matching with support for ‘?’ and ‘*’.

‘?’ Matches any single character. ‘*’ Matches any sequence of characters (including the empty sequence).

The matching should cover the entire input string (not partial).

The function prototype should be:

bool isMatch(const char s, const char p)

Some examples:

isMatch("aa","a") → false
isMatch("aa","aa") → true
isMatch("aaa","aa") → false
isMatch("aa", "*") → true
isMatch("aa", "a*") → true
isMatch("ab", "?*") → true
isMatch("aab", "c*a*b") → false

解析：

这道题目其实是Regular Expression Matching的简化版，在这里’?’相当于那边的’.’，而’*‘相当于那边的’.‘，因为这里’*’就可以代替任何字符串，不需要看前面的字符，所以处理起来更加简单。不需要用回溯法，而是直接顺序扫描即可。

？表示可以匹配任意单个字符，*表示可以匹配0~n个任意连续字符。

?很好匹配，而 * 可能需要匹配，也可能直接跳过，这取决于 * 的下一个字符与匹配串对应字符的关系，所以当遇到的时候，要记录它的下一个字符，如果下一个字符都无法匹配，这时候需要 \字符。

用bool型二维数组dp[i][j]表示s的前i个字符和p的前j个字符是否匹配。dp[0][0]为true，dp[0][j]的值取决于p中j-1位置是否为‘*’以及与j-1时的情况是否匹配。

  1. 当p[j]等于‘?’或者s[i]等于p[j]时，则dp[i][j]的值取决于dp[i-1][j-1]，即为s的前一位置和p的前一位置是否匹配；

  2. 当p[j]等于‘*’时，如果该‘*’可以匹配s中的0个或者多个字符，分别对应dp[i][j-1]，即s的当前位置和p的前一位置是否匹配，以及dp[i-1][j-1]，即s的前一位置和p的前一位置是否匹配。因为'*‘可用可不用.

由于判断当前能否匹配的时候，只是用到了j和j-1时的dp值，因此可以申请一维数组dp[i]，表示s的前i个字符和p的前j个字符是否匹配。

  1. 当p[j]不是‘*’的时候，只要判断如果当前s[i-1]和p[j-1]上的字符是否匹配，并且dp[i-1]是否为true.从后往前遍历s，这样dp[i-1]就是j-1时的值。

  2. 当p[j]是‘*’的时候，因为‘*’可以匹配任意字符串，所以在前面的dp[i]只要有true，那么剩下i后面就都是true了。

算法的时间复杂度因为是两层循环，所以是O(m*n), 而空间复杂度只用一个一维数组，所以是O(n)，假设s的长度是n，p的长度是m。

class Solution {
public:
    bool isMatch(string s, string p) {
         int ls = s.size(), lp = p.size();
         vector<bool> dp(ls + 1, false);
         dp[0] = true;
         for(int j = 1; j <= lp; ++ j)
         {
             if(p[j - 1]!='*')
                 for(int i = ls; i > 0; -- i)
                     dp[i] = dp[i - 1] && ('?' == p[j - 1] || s[i - 1] == p[j - 1]);
             else
             {
                 int i;
                 for(i = 0; i <= ls && !dp[i]; ++ i);	//找到第一个true的位置
                 for( ; i <= ls; ++ i)
                     dp[i] = true;
             }
             dp[0] = dp[0] && '*' == p[j - 1];
         }
 
         return dp[ls];
    }
};

补充:

递推做法

扫描匹配串，发现如果出现，pstar记录模板串中的下一位置，starmatch记录匹配串中与*对应的位置，并且模板串指针ptr后移，看之后的ptr是否能匹配，如果不能匹配（如果相邻的下一个也是*，那么也算匹配），那么需要让*和当前匹配串中的字符匹配

以 p = “cabc”，s = “cddabbac”为例。

对于p = “cabc”，我们可以猜想出它可以匹配的s应该长成这样： “c….ab…..c”，省略号表示0到任意多的字符。我们发现主要就是p的中间那个”ab”比较麻烦，一定要s中的’ab’来匹配，因此只要s中间存在一个”ab”，那么一切都可以交给后面的’*’了。

所以说，当我们挨个比较p和s上的字符时，当我们遇到p的第一个’*’，我们实际只需要不断地在s的剩余部分找和’ab’匹配的部分。

换言之，我们可以记录下遇到*时p和s的位置，记为pstar和starmatch，然后挨个继续比较*(++p)和*(++s)；如果发现*ptr != *str，就回溯回去，ptr = pstar，s = starmatch, ++starmatch；直到比较到末尾，或者遇到了下一个’*‘，如果遇到了下一个’‘，说明 “ab”部分搞定了，下面的就交给第二个’\‘了；如果ptr和str都到末尾了，那么就返回true；如果到末尾了既没遇到新的’*‘，又还存在不匹配的值，press也已经到末尾了，那么就返回false了。

暴力法在搜索途中就能发现不匹配的字符串，终止搜索；而DP需要在遍历完所有子问题后才能给出解，所以暴力方法快得多。

class Solution {
public:
bool isMatch(string s, string p) {
  int str=0,ptr=0,pstar=-1,starmatch=-1;
  //pstar表示p中 * 的下一个位置。
  while(str<s.length())//扫描匹配字符串
  {
      if(s[str]==p[ptr]||p[ptr]=='?')// ? 匹配单个字符
      {
         str++;
         ptr++;
      }
      else if(p[ptr]=='*')//*匹配多个字符（这次是更新ptr和starmatch，下一次才用*和s匹配）
      {
         pstar=++ptr;//*的下一个位置（*可以选择不用，下次循环尝试下个位置和str匹配）
         starmatch=str;//表示和*匹配的位置
      }
      else if(pstar>-1)//s和p无法匹配而且当前不是*，只能依靠之前的*匹配（*匹配任何字符序列）
      {
         ptr=pstar;//*和当前字符ptr匹配，ptr更新为*的下一个位置
         str=++starmatch;//str更新为下一个位置
      }
      else
          return false;
  }
  while(p[ptr]=='*')ptr++;//跳过*（对应字符可以不存在）
  return p[ptr]=='\0';
}
};

LeetCode 10. Regular Expression Matching#

Implement regular expression matching with support for ‘.’ and ‘*’.

‘.’ Matches any single character. ‘*’ Matches zero or more of the preceding element.

The matching should cover the entire input string (not partial).

The function prototype should be: bool isMatch(const char s, const char p)

Some examples:

isMatch("aa","a") → false
isMatch("aa","aa") → true
isMatch("aaa","aa") → false
isMatch("aa", "a*") → true
isMatch("aa", ".*") → true
isMatch("ab", ".*") → true
isMatch("aab", "c*a*b") → true

解析：

比较与Wildcard Matching的关系。

在Wildcard Matching中的’?’与此题的’.’一致。

但是Wildcard Matching中’‘本身代表通配符，而此题’‘代表前一个字符重复0或若干次。

DP方法

如果s [0..i）与p [0..j）匹配，则将状态P [i] [j]定义为真，否则为假。（不包括i和j位置，因为规划每个字符的时候要看后面一个字符，用右开区间更好）

class Solution {
public:
bool isMatch(string s, string p) {//p是原串 s是匹配串
    int m = s.size(), n = p.size();
    vector<vector<bool>> dp(m + 1, vector<bool>(n + 1,false));
    dp[0][0] = true;
    for (int i = 0; i <= m; i++)
        for (int j = 1; j <= n; j++){
            if (p[j - 1] == '*')//如果前一个字符是*
                dp[i][j] = dp[i][j - 2] ||       //不考虑带*字符，直接跳过
                    (i > 0 && dp[i - 1][j] && (s[i - 1] == p[j - 2] || p[j - 2] == '.'));
            //考虑带*字符，判断i位元素和带*元素是否相等
            else//如果前一个字符不是*，正常判断
                if (i > 0 && (s[i-1] == p[j-1] || p[j-1] == '.'))
                    dp[i][j] = dp[i - 1][j - 1];
        }
    return dp[m][n];
}
};

补充:

递归方法解题过程如下：

1、考虑特殊情况即*s字符串或者*p字符串结束。

（1）*s字符串结束，要求*p也结束或者间隔‘*’ （例如p=”a\b*c*……”），否则无法匹配

（2）*s字符串未结束，而*p字符串结束，则无法匹配

2、*s字符串与*p字符串均未结束

（1）*(p+1)字符不为’*‘，则只需比较*s字符与*p字符，若相等则递归到*(s+1)字符串与*(p+1)字符串的比较，否则无法匹配。

（2）*(p+1)字符为’*‘，则*p字符可以匹配*s字符串中从0开始任意多（记为i）等于*p的字符，然后递归到*(s+i+1)字符串与*(p+2)字符串的比较，

只要匹配一种情况就算完全匹配。

class Solution {
public:
    bool isMatch(string ss, string pp) {
        if(pp.size()==0) 
            return ss.size()==0;
        char *s=&ss[0];
        char *p=&pp[0];
        if (*(p + 1) != '*') {//如果下一个不是*
            if (*p == *s || (*p == '.' && *s != '\0'))//匹配或者是任意字符
                return isMatch(s + 1, p + 1);
            else
                return false;
        } else { //如果下个是*
            while (*p == *s || (*p == '.' && *s != '\0')) {
                //和当前字符相等的有2种可能：是下一个需匹配的字符或者是当前字符的合法重复,所以要每次跳过一个相等字符都进行一下后面的匹配
                if (isMatch(s, p + 2))
                    return true;
                s++;
            }
            return isMatch(s, p + 2);//没有相等字符，当前 X* 字符不存在，直接跳过
        }
    }
};
