字符串交错组成

August 22, 2017

LeetCode 97. Interleaving String#

Given s1, s2, s3, find whether s3 is formed by the interleaving of s1 and s2.

For example,

Given:

s1 = “aabcc”,

s2 = “dbbca”,

When s3 = “aadbbcbcac”, return true.

When s3 = “aadbbbaccc”, return false.

解析：

像这种判断能否按照某种规则来完成求是否或者某个量的题目，很容易会想到用动态规划来实现。

先说说维护量，res[i][j]表示用s1的前i个字符和s2的前j个字符能不能按照规则表示出s3的前i+j个字符，如此最后结果就是res[s1.length()][s2.length()]，判断是否为真即可。接下来就是递推式了。

假设知道res[i][j]之前的所有历史信息，我们怎么得到res[i][j]。可以看出，其实只有两种方式来递推，一种是选取s1的字符作为s3新加进来的字符，另一种是选s2的字符作为新进字符。而要看看能不能选取，就是判断s1(s2)的第i(j)个字符是否与s3的i+j个字符相等。如果可以选取并且对应的res[i-1]j也为真，就说明s3的i+j个字符可以被表示。这两种情况只要有一种成立，就说明res[i][j]为真，是一个或的关系。所以递推式可以表示成

f[i][j] = (f[i][j - 1] && s2[j - 1] == s3[i + j - 1]) || (f[i - 1][j] && s1[i - 1] == s3[i + j - 1]);

时间上因为是一个二维动态规划，所以复杂度是O(m*n)，m和n分别是s1和s2的长度。最后就是空间花费，可以看出递推式中只需要用到上一行的信息，所以我们只需要一个一维数组就可以完成历史信息的维护，保留j所在的第二维。每一次循环j之前都要重新维护f[0]，即f[i][0]的初始值。

class Solution {
public:
bool isInterleave(string s1, string s2, string s3) {
    if (s1.length() + s2.length() != s3.length())
        return false;
    if (s1.length() < s2.length())//固定s1的长度大于s2
        return isInterleave(s2, s1, s3);
    vector<bool> f(s2.length() + 1);//取较短的字符串的长度
    f[0]=1;
    for (int i = 1; i <= s2.length(); ++i)//关于s2的dp边界
        f[i] = s2[i - 1] == s3[i - 1] && f[i - 1];
    for (int i = 1; i <= s1.length(); ++i) {
        f[0] = s1[i - 1] == s3[i - 1] && f[0];//关于s1的dp边界。
        for (int j = 1; j <= s2.length(); ++j)
    //为了更加优化，把短的字符串放在内层循环，这样就可以只需要短字符串的长度即可
            f[j] = (s1[i - 1] == s3[i + j - 1] && f[j])|| (s2[j - 1] == s3[i + j - 1] && f[j - 1]);
    }
    return f[s2.length()];
}
};
