LeetCode 274. H-Index

Given an array of citations (each citation is a non-negative integer) of a researcher, write a function to compute the researcher’s h-index.

According to the definition of h-index on Wikipedia: “A scientist has index h if h of his/her N papers have at least h citations each, and the other N − h papers have no more than h citations each.”

For example, given citations = [3, 0, 6, 1, 5], which means the researcher has 5 papers in total and each of them had received 3, 0, 6, 1, 5 citations respectively. Since the researcher has 3 papers with at least 3 citations each and the remaining two with no more than 3 citations each, his h-index is 3.

Note: If there are several possible values for h, the maximum one is taken as the h-index.

解析：

要确定一个人的h指数非常容易，到SCI网站，查出某个人发表的所有SCI论文，让其按被引次数从高到低排列，往下核对，直到某篇论文号大于该论文被引次数，那个序号就是h指数（序号从0开始）

也就是在按值降序排列的情况下，找一个元素，该元素的下标大于该元素的值。

因为值减少，下标增加，二分查找不合理，所以直接扫描即可。

class Solution {  
public:  
int hIndex(vector<int>& citations) {  
    sort(citations.begin(),citations.end(),greater<int>());  //从大到小排序
    for(int index=0;index<citations.size();index++)  
    {  
        if(index>=citations[index])  
            return index;  
    }  
    return citations.size();  
}  
};

LeetCode 275. H-Index II

Follow up for H-Index: What if the citations array is sorted in ascending order? Could you optimize your algorithm?

解析：

数组已经升序排列,某元素i在降序排列中的实际下标为len-i。

值增加，下标也减少。找一个元素，该元素的下标大于该元素的值。符合二分查找，可以用二分查找提速。

class Solution {
public:
int hIndex(vector<int>& citations) {
    int left=0, len = citations.size(), right= len-1,  mid;
    while(left<=right)
    {
        mid=left+ (right-left)/2;
        if(citations[mid] >= (len-mid)) 
            right = mid - 1;
        else left = mid + 1;
    }
    return len - left;
}
};
